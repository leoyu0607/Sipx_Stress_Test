## 1. 下載與安裝

### 選擇適合平台的版本

| 平台 | 安裝版（推薦） | 免安裝攜帶版 |
|------|--------------|------------|
| Windows x64 | `sipress-gui-windows-x86_64-installer.msi` 或 `*-setup.exe` | `sipress-gui-windows-x86_64-portable.exe` |
| Linux x64 | `sipress-gui-linux-x86_64-installer.deb` | `sipress-gui-linux-x86_64-portable.AppImage` |
| macOS ARM64 | `sipress-gui-macos-arm64-installer.dmg` | `sipress-gui-macos-arm64-portable` |

### 安裝版 vs 攜帶版差異

- **安裝版**：向作業系統登錄程式（開始選單捷徑、解除安裝程式）。適合日常桌面使用。
- **攜帶版**：單一自包含執行檔，複製到任何位置即可執行，不會寫入系統設定。適合伺服器或共用機器。

### Windows

1. 下載 `.msi` 或 `-setup.exe` 安裝程式並執行。
2. 依照精靈步驟：接受授權、選擇安裝位置、點選「安裝」。
3. 從開始選單或桌面捷徑啟動 **sipress**。

> **攜帶版**：下載 `*-portable.exe`，直接雙擊執行，無需安裝。

### Linux

```bash
# Debian / Ubuntu — 安裝版
sudo dpkg -i sipress-gui-linux-x86_64-installer.deb

# AppImage — 攜帶版（免安裝）
chmod +x sipress-gui-linux-x86_64-portable.AppImage
./sipress-gui-linux-x86_64-portable.AppImage
```

### macOS

1. 開啟 `.dmg` 檔案，將 **sipress** 拖曳到「應用程式」資料夾。
2. 從 Launchpad 或 Spotlight 搜尋啟動。

> **首次啟動**：macOS 可能封鎖未簽章的程式。請至**系統設定 → 隱私權與安全性**，點選**「仍要開啟」**。
> 或在終端機執行：`xattr -c sipress-gui-macos-*-portable`

## 3. 選擇壓測模式

sipress 提供兩種壓測角色，在 Sidebar 上方的「📞 民眾端 / 🎧 座席端」分頁切換：

| 模式 | 行為 | 適用場景 |
|------|------|---------|
| **📞 民眾端**（Caller）| 主動發起 INVITE 撥號到接入號 | 壓測中繼線、外撥能力、IVR 路由 |
| **🎧 座席端**（Agent）| REGISTER 後等待來電並自動接聽 | 壓測座席註冊、來電分配（ACD）、座席接聽延遲 |

**重要**：兩種模式完全獨立。切到座席端後按「開始測試」，Rust 後端會跑 `AgentEngine`（不是 Caller engine）。CLI Preview 會反映目前選擇的模式。

## 4. 設定測試參數

### 目標設定（兩個模式共用）

| 欄位 | 範例 | 說明 |
|------|------|------|
| SIP 伺服器 | `192.168.1.100:5060` | 目標 SIP Proxy 的 IP 與 Port |
| 傳輸層 | `UDP` / `TCP` / `TLS` | 須與伺服器設定一致 |
| 本機 Port | `5080` | 留空則由作業系統自動分配 |
| 持續時間（秒）| `60` | **設為 0 = 不限時間**，需搭配總通數或手動停止 |

### 民眾端設定（Caller mode）

| 欄位 | 範例 | 說明 |
|------|------|------|
| 交換機接入號 | `4008001234` | From 標頭中的主叫號碼 |
| 被叫號碼 | `8001` 或留空 | 留空時用「前綴 + 隨機尾數」分散負載 |
| 併發數量 | `100` | 最大同時進行中通話數 |
| 呼叫頻率 CPS | `10` | 每秒發起的新通話數 |
| 總測試通數 | `500` | **設為 0 = 不限通數**，依持續時間決定結束 |

### 座席端設定（Agent mode）

| 欄位 | 範例 | 說明 |
|------|------|------|
| 預設 SIP Domain | `sip.example.com` | 用於 From/To URI；空字串時自動取伺服器 IP |
| 帳號清單 | （見下節）| 透過拖曳 CSV / 快速生成 / 手動輸入 |

> **停止條件優先順序**
> 1. 手動按下「■ 停止測試」
> 2. 總通數達到上限（若 > 0，且目前通話全部結束）
> 3. 持續時間到期（若 > 0）
>
> 若兩者皆設為 0，測試不會自動停止，必須手動停止。

### 音訊設定（選用，僅民眾端）

啟用「播放音檔」開關後：

1. 點選「瀏覽」開啟系統檔案選擇器。
2. 選擇音檔（支援 `.wav` PCM16/A-law/μ-law、`.al/.alaw`、`.ul/.ulaw/.pcmu`、`.raw`）。
3. 程式會**自動轉換成 G.711A（PCMA）** 後傳送，無需先用 ffmpeg 處理。
4. SDP offer 只宣告 `RTP/AVP 8`（PCMA），避免伺服器選到 PCMU。
5. 每通接通的通話都會傳送 RTP 音訊封包並計算 MOS 聲音品質分數。

停用音訊為純 SIP 信令測試（速度較快、CPU 使用率較低）。

## 5. 座席端：帳號管理與註冊

切到「🎧 座席端」分頁後，Sidebar 中段會出現帳號管理區。

### 5.1 新增帳號的三種方式

#### A) 拖曳 / 上傳 CSV / TXT 檔

支援三種格式（首行若是標頭會自動跳過）：

```csv
extension,username,password,domain
1001,user1001,pass1001,sip.example.com
1002,user1002,pass1002,sip.example.com
```

```
1001:pass1001:sip.example.com
1002:pass1002:sip.example.com
```

```
1001 user1001 pass1001 sip.example.com
1002 user1002 pass1002 sip.example.com
```

`domain` 欄位可省略，會用「預設 SIP Domain」遞補。

#### B) 「+ 快速生成帳號」按鈕

開啟對話框輸入：起始分機號、產生數量、帳號前綴、密碼模式（同一組密碼 / 密碼=分機號 / 自訂模板含 `{ext}`）、Domain。

#### C) 手動編輯

帳號清單表格中每格都可以直接點擊編輯（分機 / 帳號 / 密碼）。

### 5.2 帳號狀態 badge

每個帳號加入時會**立即觸發 SIP REGISTER**（含 Digest 認證重送）做即時驗證：

| Badge | 顏色 | 含義 |
|-------|------|------|
| `—` | 灰 | idle（剛建立還沒註冊） |
| `…` | 黃 | registering（REGISTER 中） |
| `REG` | 綠 | 已註冊成功，可開始測試 |
| `ERR` | 紅 | 失敗：認證錯誤 / 拒絕 / 逾時。底部 Event Log 會寫具體 SIP 狀態碼 |

註冊成功後 Sidebar 底部會顯示 `已註冊 N / 總數`。

### 5.3 重新註冊與清除

- **重新註冊**：對所有非 `REG` 的帳號重發 REGISTER，常用於修改密碼後或網路恢復。
- **匯出 CSV**：把目前清單存成 CSV，下次可直接拖回來。
- **清除全部**：清空帳號清單（不發 REGISTER Expires=0；伺服器端的舊註冊會自然過期）。

### 5.4 開始測試後的行為

按「▶ 開始測試」後：

1. Rust 後端為**每個帳號**獨立建立 UDP socket 並再做一次 REGISTER（這次是壓測用的常駐連線，不同於上面的即時驗證）
2. 每個帳號自動在 `Expires/2` 計時器到期時 re-REGISTER 維持註冊
3. 收到伺服器送來的 INVITE → 自動回 100 Trying → 200 OK + SDP（PCMA） → 等 ACK → 通話建立
4. 收到 BYE → 200 OK，計入完成通數
5. 也會處理 RE-INVITE（Session-Expires）/ CANCEL / OPTIONS
6. 測試結束時對每個帳號發 REGISTER Expires=0 解除註冊

## 6. 執行測試與讀取即時指標

### 啟動測試

1. 填寫「SIP 伺服器」與「交換機接入號」（最少必填）。
2. 點選標題列的 **▶ 開始測試**。
3. 指標列每秒自動更新：

| 指標 | 意義 | 健康參考值 |
|------|------|----------|
| **CPS** | 每秒實際發起的通話數 | 接近設定值 |
| **CONCUR** | 目前進行中的通話數 | ≤ 設定的併發上限 |
| **SUCCESS** | 累計成功接通通數（200 OK） | 越高越好 |
| **FAILED** | 累計失敗通數（含逾時） | 越低越好 |
| **QUEUED** | 目前等待回應的 INVITE 數量 | 接近 CONCUR |
| **ASR** | 接通率（%） | ≥ 85% 優、≥ 70% 普通 |
| **ERR%** | 失敗率（%） | ≤ 5% 正常 |
| **PDD** | 平均撥後延遲（ms） | < 200ms 正常 |

### 圖表標籤頁

| 標籤 | 顯示內容 |
|------|---------|
| CPS | 每秒發起量趨勢 |
| CONC | 併發數趨勢 |
| ASR | 接通率趨勢 |
| CCR | 完成率趨勢 |
| PDD | 延遲趨勢 |
| FAIL | 失敗累計趨勢 |
| MOS | 聲音品質趨勢（需啟用 RTP） |

### 右側面板

- **通話狀態**：INVITE → 100 Trying → 180 Ringing → 200 OK → ACK → BYE 各狀態比例
- **RTP 聲音品質**（啟用音訊後顯示）：
  - **MOS**：1.0–5.0，≥ 4.0 優、≥ 3.0 普通、< 2.5 差
  - **掉包率（%）**：< 1% 優、< 3% 普通、≥ 3% 差
  - **Jitter（ms）**：< 30ms 優、< 60ms 普通、≥ 60ms 差
- **回應碼統計**：每個 SIP 回應碼的累計次數
- **最後通話 SIP 流程**：最近一通通話的時序圖

### 停止測試

點選 **■ 停止測試**，測試立即停止，並顯示最終結果。

## 7. 查看測試結果

測試結束後（時間到、通數上限、或手動停止），指標固定為最終值，日誌最後一行顯示摘要。

### 匯出報告

測試完成後，TitleBar 右側的三個匯出按鈕均可點擊：

| 按鈕 | 格式 | 用途 |
|------|------|------|
| ↓ JSON | `sipress_<時間戳>.json` | CI/CD 整合、程式讀取 |
| ↓ CSV | `sipress_<時間戳>.csv` | Excel、Python 分析 |
| ↓ HTML | `sipress_<時間戳>.html` | 視覺化報告（含環形圖、延遲長條圖、RTP 品質），可用瀏覽器離線開啟 |

> **注意**：`↓ HTML` 按鈕在測試**執行中**或**尚未開始**時為灰色（disabled），需等測試完成或手動停止後才能使用。

## 8. CLI 使用方式

適合腳本化或無桌面的伺服器環境。

### 基本範例

```bash
# 最基本：100 併發、10 CPS、持續 60 秒
./sipress -s 192.168.1.100:5060 \
  --number 4008001234 \
  -c 100 --cps 10 -d 60

# 不限時間，共測試 1000 通
./sipress -s 192.168.1.100:5060 \
  --number 4008001234 \
  -c 50 --cps 5 -d 0 --max-calls 1000

# 啟用 RTP 音訊
./sipress -s 192.168.1.100:5060 \
  --number 4008001234 \
  -c 100 --cps 10 -d 60 \
  --audio /path/to/sample.wav

# 輸出 JSON 報告
./sipress -s 192.168.1.100:5060 \
  --number 4008001234 \
  -c 100 --cps 10 -d 60 --format json
```

### 完整參數列表

```
--server / -s <位址>       SIP 伺服器 host:port
--transport <協定>          udp | tcp | tls（預設：udp）
--number <號碼>             主叫號碼（From 標頭）
--concurrent / -c <N>      最大同時通話數（預設：100）
--cps <N>                  每秒通話數（預設：10）
--duration / -d <秒>        測試時長，0 = 不限（預設：60）
--max-calls <N>            總通話上限，0 = 不限
--call-duration <秒>        單通通話持續時間（預設：30）
--invite-timeout <秒>       INVITE 逾時（預設：8）
--audio <檔案>              WAV/PCM 音檔（啟用 RTP）
--rtp-base-port <Port>     RTP 起始 Port（預設：10000）
--format <格式>             table | json | csv（預設：table）
--logs-dir <目錄>           Log 輸出目錄（預設：logs）
--tui                      啟用即時儀表板介面
```

## 9. 常見問題排除

### macOS 顯示「無法打開，因為無法驗證開發者」

```bash
xattr -c sipress-gui-macos-*-portable
```
或至**系統設定 → 隱私權與安全性**，點選**「仍要開啟」**。

### 完全沒有通話被發出

- 用 `ping` 或 `nc -u <host> <port>` 確認伺服器可連線
- 確認防火牆允許對外 UDP/TCP Port 5060
- 確認主叫號碼格式符合伺服器要求

### 民眾端：所有通話回傳 403 / 401（認證失敗）

民眾端模式的 INVITE 不帶認證標頭。請在 SIP 伺服器端將測試來源 IP 加入白名單，或對該測試 Trunk 停用認證。
（座席模式有完整 Digest 認證實作，可正常註冊有密碼的帳號）

### 座席端：帳號 badge 一直是 `ERR`

依 Event Log 中具體訊息分類處理：

| Event Log 訊息 | 原因 | 處理 |
|---|---|---|
| `認證失敗 (SIP 401)` | 帳號 / 密碼錯 | 確認密碼大小寫、特殊字元 |
| `認證失敗 (SIP 403)` | 帳號被禁用或不存在 | 在交換機端確認帳號 enabled |
| `註冊被拒 (SIP 503)` | 伺服器忙線或註冊資源滿 | 降低同時新增的帳號數量 |
| `註冊被拒 (SIP 423)` | 要求的 Expires 太短 | 通常 sipress 用 3600s，極少觸發 |
| `REGISTER 等待回應逾時` | 網路不通或防火牆擋 | `ping` server，確認 5060 UDP 可通 |
| `無法解析 Digest challenge` | 伺服器用非標準格式或非 Digest | 檢查 SIP log 中 `WWW-Authenticate` 標頭 |

### 座席端：通話接通幾秒就被掛斷

可能原因：

1. 交換機檢查 RTP 流量，沒收到對端音訊 → 砍掉通話。Phase 2 座席模式目前**只回應 SIP，不真的收/送 RTP**，下一版會補。
2. Session-Expires 過短，交換機要求頻繁 RE-INVITE → 已自動處理，若仍發生請檢查 SIP log 中 RE-INVITE 是否有回應。

### 座席端：按開始測試後好像跑成民眾端

從 GUI 0.1 起，「開始測試」會依 Sidebar 上方的分頁切換真正執行對應引擎。若懷疑沒切換：

- 看 Event Log 第一行是否寫 `[agent mode]` 或 `[caller mode]`
- 看 SIP log（`logs/YYYYMMDD_HHMMSS_agent.sip.log`）首先送出的是 `REGISTER`（座席）還是 `INVITE`（民眾）
- 重新點選一次「🎧 座席端」分頁確保 active 狀態

### 大量通話失敗

| 錯誤碼 | 原因 | 處理方式 |
|--------|------|---------|
| 486 Busy Here | 伺服器過載 | 降低 CPS 或併發數 |
| 503 Service Unavailable | 伺服器離線 | 確認伺服器狀態 |
| 408 Timeout / ERR% ≈ 100% | 伺服器不存在或網路不通 | 這是正確行為；所有通話會在 `invite_timeout` 後標記為逾時失敗 |

### MOS 顯示 0 或 N/A

1. 確認已啟用「播放音檔」並選擇有效的音檔。
2. 確認伺服器接受 RTP 媒體流（有接通的通話才會計算 MOS）。
3. 座席模式目前不收/送 RTP，因此 MOS 永遠是 N/A（這是預期行為）。

### 接通後伺服器錄音檔很短只有幾秒

過去版本：sipress 不處理伺服器送來的 RE-INVITE（Session-Expires），約 9 秒後伺服器以為失聯就掛斷 → 錄音檔只有開頭。

現在已修補：
- 收到 RE-INVITE 自動 200 OK + SDP 保活
- 收到伺服器 BYE 也會自動回 200 OK 並結束統計

若仍遇到，請檢查 `logs/*_agent.sip.log` 中是否有 `收到 RE-INVITE...回應 200 OK` 的紀錄。

### GUI 顯示空白視窗

刪除設定後重新啟動：
- Windows：刪除 `%APPDATA%\com.leozh.sipress`
- Linux：刪除 `~/.config/com.leozh.sipress`

### 持續時間設為 0 但測試不停止

`duration = 0` 需搭配以下其中一項才能自動停止：
- 設定「總測試通數」為 > 0
- 手動按下「■ 停止測試」

### Log 檔案位置
